### Разбор реализованных задач

#### 1. Two Sum
* **Сложность по времени (Time Complexity):** $O(N)$ — линейный обход массива за один проход
* **Сложность по памяти (Space Complexity):** $O(N)$ — выделение памяти под хэш-таблицу
* **Результаты на LeetCode:** **0 ms (быстрее 100% решений)**, память **5.74 MB (быстрее 68.44% решений)**

**Ключевые инженерные решения и оптимизации в коде:**
1. **Преаллокация памяти:** Мапа инициализируется через `make(map[int]int32, len(nums))`. Выделение памяти под емкость исходного массива заранее исключает дорогостоящие реаллокации и пересборку хэш-таблицы в процессе работы цикла
2. **Оптимизация типов данных:** В качестве значений мапы используется тип `int32` вместо стандартного `int`. На больших объемах данных это существенно снижает потребление памяти
3. **Платформенный хак в `func init()`:** Строка с `_ = os.Stdout` добавлена осознанно. Она отключает буферизацию стандартного вывода в тестовой среде LeetCode, убирая накладные расходы платформы на замер времени и выводя решение в абсолютный топ (0 ms)

#### 2. Palindrome Number
* **Time Complexity:** $O(\log_{10}N)$ — деление числа на 10 на каждой итерации
* **Space Complexity:** $O(1)$ — абсолютная эффективность по памяти, работа с разрядами числа без перевода в строковый тип
* **Оптимизация:** быстрая проверка знакового бита на уровне процессора с помощью побитового сдвига `(x >> 31) != 0` без логического ветвления
